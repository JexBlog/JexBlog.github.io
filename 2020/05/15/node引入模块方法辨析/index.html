JS导出引入模块方法辨析

Posted on 2020-05-15

列举常见导出引入模块的方式

导出： export export default module.exports exports

引入：require import

导出

module.exports 和 exports

在node的模块里，可以直接调用 exports 和 module 两个全局变量，但 exports 是 module.exports 的一个引用。
require 引用模块后，返回的是 module.exports 而不是 exports。

如果 exports = function(){}; 这样就是重新给 exports 赋值，它将不再是 module.exports 的引用，二者将无任何联系。

避免混淆，建议不用 exports 直接用 module.exports。

export default 和 export

export default 与 export 均可用于导出常量、函数、文件、模块等。

在一个文件或模块中，export、import 可以有多个，export default 仅有一个。

通过 export 方式导出，在导入时要加 { }，export default 则不需要。

export 能直接导出变量表达式，export default 不行。

导入

require

require 是运行时，可以放在任何地方。require 相当于 module.exports 的传送门，module.exports 后面的内容是什么，require的结果就是什么。

import

import 是编译时，必须放在文件开头。它不会将整个模块运行后赋值给某个变量，而是只选择import 的接口进行编译，这样在性能上比 require 好一些。
